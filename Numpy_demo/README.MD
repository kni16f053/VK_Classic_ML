# Домашнее задание №1: Реализация алгоритмов с использованием NumPy

В данной работе представлены реализации различных алгоритмов обработки данных с использованием библиотек `NumPy`, `Matplotlib` и `imageio`. Для каждого задания выполнена реализация тремя способами: "чистым" Python с циклами и двумя векторизованными способами с помощью NumPy. Проведено сравнение производительности и построены графики зависимости времени выполнения от размера входных данных.

## Общие вспомогательные функции

Для всех заданий используются общие функции для замеров времени и построения графиков:

*   `timeitTest(Func, *args)` — замеряет среднее время выполнения функции.
*   `plot3TimeTests(...)` и `plot2TimeTests(...)` — строят графики времени выполнения для двух или трех функций.
*   `plot2TimeRegression(...)` — строит график и линию линейной регрессии для двух функций.

## Общие выводы

По результатам работы можно сделать следующие заключения:
1.  **Использование циклов на Python (`for`, `while`) для обработки числовых массивов крайне неэффективно.** Эту стратегию следует использовать только для очень маленьких объемов данных или когда векторизация невозможна.
2.  **Векторизованные операции NumPy позволяют достичь значительного ускорения за счет выполнения операций на низкоуровневых, оптимизированных языках (C/Fortran).**
3.  **Среди векторизованных решений важно выбирать наиболее подходящее.** В некоторых случаях более простая и читаемая реализация (например, с `np.diagonal` или `np.where`) может быть не только понятнее, но и быстрее.
4.  **Анализ производительности с помощью регрессии подтверждает линейную сложность `O(n)` для эффективных алгоритмов** и позволяет наглядно сравнить их константные множители.
