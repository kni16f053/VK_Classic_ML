# ДЗ №2 — Логистическая регрессия. Предсказание погоды

## Что сделано
- Загружены и предобработаны метеорологические данные (142 193 записи)
- Проведён глубокий анализ пропусков и их заполнение:
  - Разбивка даты на числовые признаки (год, месяц, день)
  - Преобразование направлений ветра в радианы
  - Групповое заполнение пропусков средними значениями по локации и месяцу
  - Удаление признаков с систематическими пропусками (`Evaporation`, `Sunshine`, `Cloud9am`, `Cloud3pm`)
- Кодирование категориального признака `Location` с помощью `BinaryEncoder`
- Построены и сравнены 4 модели для предсказания `RainTomorrow`:
  1. **Логистическая регрессия** (sklearn) — accuracy 85.03%
  2. **K-ближайших соседей** — лучший accuracy 83.65% (k=9)
  3. **Наивный Байесовский классификатор** — accuracy 82.86%
  4. **Логистическая регрессия** (собственная реализация) — accuracy до 76.19%

## Лучший результат
- **Алгоритм:** Логистическая регрессия (sklearn.linear_model.LogisticRegression)
- **Accuracy на тесте:** 85.03%
- **Accuracy на трейне:** 83.64%
- **Время обучения:** 0.36 секунды

## Освоенные техники
- Анализ пропусков (missingno)
- Кодирование категориальных признаков
- Преобразование направлений в числовые значения (радианы)
- Групповая агрегация для заполнения пропусков
- Стандартизация признаков
- Реализация логистической регрессии с нуля (градиентный спуск, logloss)
- Сравнение моделей по качеству и времени выполнения

## Выводы
Лучшие результаты показала библиотечная логистическая регрессия — она проста, быстра и хорошо справляется с задачей. Собственная реализация уступает по качеству из-за отсутствия L2-регуляризации и продвинутых оптимизаций, но демонстрирует работоспособность подхода.
